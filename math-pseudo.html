Lesson 01 · Pseudocode & Flowcharts

What is an Algorithm?

An algorithm is a finite, ordered set of unambiguous instructions that solves a problem. It's the idea behind code — language-independent, platform-independent, timeless. Every program you write executes an algorithm.

📋 Five Properties of a Good Algorithm
  • Finiteness — It must terminate after a finite number of steps
  • Definiteness — Every step is clearly defined, no ambiguity
  • Input — Zero or more inputs are provided
  • Output — One or more outputs are produced
  • Effectiveness — Every step is basic enough to be carried out exactly
🍳 Everyday Algorithm: Making Tea

1. Boil water. 2. Place teabag in cup. 3. Pour boiling water over teabag. 4. Wait 3 minutes. 5. Remove teabag. 6. Add milk if desired. 7. Serve. — This is an algorithm: finite, ordered, unambiguous, produces output (tea).

Why Design Before You Code?

Professional engineers don't start with code. They start with the algorithm. Writing pseudocode first lets you solve the logical problem without worrying about syntax errors, language quirks, or compiler warnings. You think clearly, then you type.

ApproachSkip DesignDesign First
Bug locationHard — logic + syntax mixedEasy — logic already verified
Language changeRewrite from scratchTranslate pseudocode to new language
Team communicationMust understand codeAnyone can read pseudocode
Complexity analysisNeeds code executionEasy to count steps in pseudocode

Pseudocode Conventions

Pseudocode is structured English that describes an algorithm step-by-step. There's no universal standard — different textbooks use different notation — but key constructs are universal.

ConstructPseudocodePython equivalent
AssignmentSET x ← 5x = 5
InputREAD namename = input()
OutputPRINT "Hello"print("Hello")
ConditionalIF x > 0 THEN … ELSE … END IFif x > 0: … else: …
While loopWHILE x < 10 DO … END WHILEwhile x < 10: …
For loopFOR i FROM 1 TO n DO … END FORfor i in range(1, n+1): …
FunctionFUNCTION name(params) … RETURN valdef name(params): return val
ALGORITHM: Find Maximum Value INPUT: list of numbers OUTPUT: the largest number FUNCTION FindMax(numbers) SET max ← numbers[0] // start with first element FOR i FROM 1 TO LENGTH(numbers) - 1 DO IF numbers[i] > max THEN SET max ← numbers[i] END IF END FOR RETURN max END FUNCTION
# Python translation: def find_max(numbers): max_val = numbers[0] for i in range(1, len(numbers)): if numbers[i] > max_val: max_val = numbers[i] return max_val
✅ Good Pseudocode Rules
  • Use UPPERCASE for keywords (IF, WHILE, RETURN)
  • Indent consistently — structure should be visually obvious
  • Be precise about data: use ← for assignment, = for comparison
  • Name things clearly: FindMax not Proc1
  • Each line = one logical action

Reading Pseudocode

What does the following pseudocode return for input n=5?

FUNCTION Mystery(n) SET result ← 0 FOR i FROM 1 TO n DO SET result ← result + i END FOR RETURN result END FUNCTION
Trace: result = 0+1+2+3+4+5 = 15. This computes the sum 1+2+…+n.

Flowchart Symbols

A flowchart is a visual representation of an algorithm. Each shape has a specific meaning — using the right symbol makes your diagram unambiguous and universally readable.

ShapeNameMeaning
⬭ Oval / Rounded rectTerminalSTART or END of the algorithm
▭ RectangleProcessAn action or computation step
◇ DiamondDecisionA YES/NO question — branches the flow
▱ ParallelogramInput/OutputRead data (input) or display result (output)
→ ArrowFlowlineDirection of flow between steps
⃝ Small circleConnectorContinues flow on another part of diagram

Each symbol follows the ISO 5807 flowchart standard.

Common Algorithms in Pseudocode

Let's look at two classic algorithms every programmer should know — Linear Search and Bubble Sort — in pseudocode, then watch them run step by step.

Linear Search

ALGORITHM: Linear Search FUNCTION LinearSearch(list, target) FOR i FROM 0 TO LENGTH(list) - 1 DO IF list[i] = target THEN RETURN i // found at index i END IF END FOR RETURN -1 // not found END FUNCTION // Time complexity: O(n) — must check up to n elements

Bubble Sort

ALGORITHM: Bubble Sort FUNCTION BubbleSort(list) FOR pass FROM 1 TO LENGTH(list) - 1 DO FOR i FROM 0 TO LENGTH(list) - pass - 1 DO IF list[i] > list[i+1] THEN SWAP list[i] WITH list[i+1] END IF END FOR END FOR RETURN list END FUNCTION // Time complexity: O(n²) — nested loops

⏱️ Time Complexity: O(n) vs O(n²)

Linear search checks each element once — O(n). Bubble sort has nested loops — O(n²). For 1,000 elements: linear search does ~1,000 checks; bubble sort does ~1,000,000. Always ask: what's the worst case? Can I do better?
