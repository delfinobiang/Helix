Arrays & Memory

An array is a contiguous block of memory where each element sits at a predictable address. This makes index access O(1) — one of the most important properties in all of computing.

📦 How Arrays Work in Memory

If an array starts at address 1000 and each int takes 4 bytes, then arr[3] is at address 1000 + (3 × 4) = 1012. The CPU computes this in one operation — no searching required.

Linked Lists

A linked list stores elements as nodes, each pointing to the next. There's no contiguous memory — nodes can be scattered anywhere. This makes insertions fast but random access slow.

🔗 Node Structure

Each node holds a value and a pointer to the next node. The last node points to None. Traversal means following pointers one by one — you can't jump to the middle directly.

class Node: def __init__(self, val): self.val = val self.next = None class LinkedList: def __init__(self): self.head = None def append(self, val): node = Node(val) if not self.head: self.head = node; return cur = self.head while cur.next: # O(n) — walk to end cur = cur.next cur.next = node def prepend(self, val): # O(1) — just update head node = Node(val) node.next = self.head self.head = node

Stacks & Queues

Stacks and queues are abstract data types that restrict how you add and remove elements. These restrictions make them perfect tools for specific algorithmic patterns.

📚 Stack: Last In, First Out (LIFO)

A stack only allows push (add to top) and pop (remove from top). Think of a stack of plates — you can only add or remove from the top. Used in: function call stack, undo/redo, expression parsing.

🚶 Queue: First In, First Out (FIFO)

A queue allows enqueue (add to back) and dequeue (remove from front). Think of a line at a shop — first person in is first served. Used in: BFS, job schedulers, event systems, print queues.

# Python: list as stack stack = [] stack.append(1) # push stack.append(2) stack.pop() # → 2 (LIFO) # Python: deque as queue (O(1) both ends) from collections import deque q = deque() q.append(1) # enqueue q.append(2) q.popleft() # → 1 (FIFO)

Hash Tables

A hash table maps keys to values using a hash function. It achieves average O(1) for insert, lookup, and delete — the magic behind Python dicts, JavaScript objects, and database indexes.

🔑 How Hashing Works

A hash function converts any key into an integer index. hash("hello") % 10 → 5 means store at slot 5. Collisions happen when two keys hash to the same slot — resolved by chaining (linked list in the slot) or open addressing.

# Python dicts are hash tables d = {} d["name"] = "Alice" # O(1) insert d["age"] = 30 # O(1) insert print(d["name"]) # O(1) lookup del d["age"] # O(1) delete # Sets are hash tables (keys only) seen = set() for x in arr: if x in seen: # O(1) — not O(n)! return x seen.add(x)
📖
Dictionaries
Python dicts, JS objects, JSON — all hash tables. The backbone of almost every real program.
🔐
Caching
Redis and Memcached store key→value in memory using hash tables for O(1) lookups at massive scale.
🗄️
Database Indexes
Hash indexes let databases find rows by primary key in O(1) without scanning the whole table.
✨
Deduplication
Sets (hash tables) solve "find duplicates" in O(n) instead of O(n²). Used everywhere in data pipelines.
