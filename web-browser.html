How Browsers Work

A browser isn't just a window to the web — it's a sophisticated runtime that parses code, builds data structures, calculates layouts, and paints pixels — all in milliseconds. Understanding this pipeline is essential for writing performant web apps.

🔄 The Critical Rendering Path
  • 1. Parse HTML → build the DOM (Document Object Model)
  • 2. Parse CSS → build the CSSOM (CSS Object Model)
  • 3. Combine → produce the Render Tree (only visible nodes)
  • 4. Layout → calculate exact position and size of every element
  • 5. Paint → fill in pixels (text, colours, images, shadows)
  • 6. Composite → combine layers on the GPU and display

This entire pipeline must complete within ~16ms to achieve 60fps. Layout and paint are expensive — CSS properties like transform and opacity skip layout/paint entirely and only trigger compositing, making them the go-to for smooth animations.

The DOM Tree

The DOM (Document Object Model) is the browser's in-memory representation of an HTML page as a tree of nodes. JavaScript can read and modify this tree to make pages dynamic — every framework (React, Vue, Angular) ultimately works through the DOM.

🌲 Node Types
  • Document node — root of the tree (document)
  • Element nodes — HTML tags (<div>, <p>, etc.)
  • Text nodes — the text content inside elements
  • Attribute nodes — element attributes (class, id, href)
  • Comment nodes — HTML comments
// Selecting elements const btn = document.getElementById('submit-btn'); const items = document.querySelectorAll('.list-item'); const first = document.querySelector('h1'); // Modifying the DOM btn.textContent = 'Loading...'; btn.classList.add('active'); btn.setAttribute('disabled', ''); // Creating and inserting elements const li = document.createElement('li'); li.textContent = 'New item'; document.querySelector('ul').appendChild(li); // Traversal el.parentElement; // parent node el.children; // child elements el.nextElementSibling; // next sibling

Event Handling

Events are how browsers communicate user actions and state changes to your JavaScript. Every click, keypress, scroll, and network response fires an event. Mastering the event system is key to building interactive applications.

📡 Event Bubbling & Capturing

When a user clicks a nested element, the event propagates in three phases:

1. Capture phase — event travels down from document to the target
2. Target phase — event reaches the clicked element
3. Bubble phase — event bubbles up from target back to document

Most handlers use the bubble phase. Call event.stopPropagation() to prevent bubbling.

// Adding event listeners btn.addEventListener('click', function(event) { event.preventDefault(); // stop default (e.g. form submit) console.log('Clicked!', event.target); }); // Event delegation — one listener for many children document.querySelector('ul').addEventListener('click', (e) => { if (e.target.matches('li')) { console.log('List item clicked:', e.target.textContent); } }); // Common events 'click', 'dblclick' // mouse 'keydown', 'keyup' // keyboard 'submit', 'input', 'change' // forms 'DOMContentLoaded' // DOM ready (before images) 'load' // page fully loaded 'scroll', 'resize' // window
🎯 Event Delegation

Instead of adding listeners to every list item, add one listener to the parent and check which child was clicked using event.target. This is more memory-efficient and works for dynamically added elements.

Developer Tools

Browser DevTools are the most powerful debugging weapon in a web developer's arsenal. Every major browser ships them — and learning to use them well will save you hours every week.

🛠️ Key DevTools Panels
  • Elements — inspect and live-edit the DOM and CSS. Right-click any element on a page → Inspect.
  • Console — run JavaScript, see errors, log values with console.log()
  • Network — see every HTTP request, response headers, timing, and payload
  • Sources — view page source, set breakpoints, debug JavaScript step-by-step
  • Performance — record and analyse rendering, scripting, and layout time
  • Application — inspect localStorage, sessionStorage, cookies, IndexedDB
  • Lighthouse — automated audit for performance, accessibility, SEO, and best practices
// Console tricks console.log('%c Big text', 'font-size:24px;color:hotpink'); console.table([{name:'Alice',age:30},{name:'Bob',age:25}]); console.group('Network'); console.log('...'); console.groupEnd(); console.time('sort'); arr.sort(); console.timeEnd('sort'); // Debugging in Sources debugger; // pauses execution here when DevTools is open // Live-editing the DOM in console document.querySelector('h1').textContent = 'Hello DevTools!';
⚡ Pro Tips
  • $0 in console refers to the last element you selected in Elements panel
  • $$('.class') is a shorthand for querySelectorAll in DevTools console
  • Use Network throttling to simulate 3G and see how slow users experience your site
  • Lighthouse is built-in — run an audit on any site to see where it loses points
  • Right-click → Override content to mock API responses without changing the server
