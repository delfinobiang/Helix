Lesson 01 · Data & AI

What is Machine Learning?

Machine learning is the field where systems learn patterns from data without being explicitly programmed. Instead of writing rules, you feed examples and the algorithm infers the rules itself.

🤖 Traditional Programming vs ML

Traditional: Data + Rules → Output
Machine Learning: Data + Output → Rules (model)

TypeDescriptionExample
SupervisedLearn from labeled examplesSpam detection, image classification
UnsupervisedFind structure in unlabeled dataCustomer clustering, anomaly detection
ReinforcementLearn by trial and rewardGame AI, robotics, recommendation
Semi-supervisedMix of labeled and unlabeledMedical imaging with few labels

Lesson 02 · Data & AI

Supervised Learning

In supervised learning, you train a model on labeled data to predict outputs for new inputs. The two main tasks are classification (discrete outputs) and regression (continuous outputs).

📦 Classification vs Regression
  • Classification: Predict a category — spam/ham, cat/dog, disease/healthy
  • Regression: Predict a number — house price, temperature, user age

Lesson 03 · Data & AI

Unsupervised Learning

Unsupervised learning finds hidden structure in unlabeled data. There are no right answers to train from — the algorithm discovers patterns on its own.

🔵 Clustering (K-Means)

Group data points into k clusters so points within each cluster are similar. Algorithm: randomly place k centroids, assign each point to nearest centroid, update centroids to cluster means, repeat until stable.

📉 Dimensionality Reduction (PCA)

Compress high-dimensional data into fewer dimensions while preserving most variance. A dataset with 100 features might be visualizable in 2D using PCA. Used for visualisation and speeding up training.

from sklearn.cluster import KMeans from sklearn.decomposition import PCA # Cluster customers into 3 groups kmeans = KMeans(n_clusters=3, random_state=42) labels = kmeans.fit_predict(X) # assigns 0, 1, or 2 # Reduce to 2D for visualisation pca = PCA(n_components=2) X_2d = pca.fit_transform(X) print(pca.explained_variance_ratio_) # how much variance kept

Lesson 04 · Data & AI

Model Evaluation

A model that gets 99% accuracy on training data but fails on new data is useless. Overfitting, underfitting, and proper evaluation metrics are the difference between research toy and real product.

⚖️ Bias–Variance Tradeoff
  • Underfitting (high bias): Model too simple — misses patterns. Train accuracy low.
  • Overfitting (high variance): Model memorises training data — fails on new data. Train high, test low.
  • Sweet spot: Good generalisation — train ≈ test, both high.
MetricFormulaUse when
Accuracy(TP+TN) / totalBalanced classes
PrecisionTP / (TP+FP)Cost of false positives is high
RecallTP / (TP+FN)Cost of false negatives is high
F1 Score2 × P × R / (P+R)Imbalanced classes
RMSE√(mean squared error)Regression tasks

Spam Filters
Gmail's ML model processes billions of emails daily. High recall catches more spam; high precision avoids false positives.
🏥
Cancer Detection
Medical ML prioritises recall — missing a cancer (false negative) is far worse than a false alarm.
🎵
Recommendations
Spotify trains models on billions of listening events to predict which songs you'll want to hear next.
🚗
Self-Driving Cars
Computer vision models classify objects at 30fps — pedestrian, car, sign — to navigate autonomously.
