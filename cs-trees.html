Lesson 01 · CS Theory

Tree Fundamentals

A tree is a connected, acyclic graph. Trees model hierarchical data — file systems, company org charts, HTML DOM, compilers' abstract syntax trees. Every tree has one root and every node has exactly one parent (except the root).

🌳 Key Terms
  • Root: The top node with no parent
  • Leaf: A node with no children
  • Height: Longest path from root to any leaf
  • Depth: Distance from root to a specific node
  • Subtree: A node plus all its descendants

Tree Height

A perfect binary tree has 3 levels (root at level 0, leaves at level 2). How many nodes does it have in total?

Level 0: 1 node. Level 1: 2 nodes. Level 2: 4 nodes. Total: 1+2+4 = 7. Formula: 2^h − 1 = 2^3 − 1 = 7.
Lesson 02 · CS Theory

Binary Search Trees

A Binary Search Tree (BST) stores values so that for every node, all left descendants are smaller, and all right descendants are larger. This property enables O(log n) search, insert, and delete on balanced trees.

🔍 BST Search Property

To find value X: if X < current node, go left. If X > current node, go right. If X = current node, found! Each step eliminates half the remaining tree — O(log n) on a balanced tree.

class BST: def insert(self, node, val): if node is None: return Node(val) if val < node.val: node.left = self.insert(node.left, val) else: node.right = self.insert(node.right, val) return node def search(self, node, val): if node is None or node.val == val: return node if val < node.val: return self.search(node.left, val) return self.search(node.right, val)

BST Search Direction

In a BST, the root is 50. You're searching for 30. Do you go left or right? Type: left or right

Lesson 03 · CS Theory

Tree Traversal

Tree traversal visits every node exactly once. The three main orderings — in-order, pre-order, post-order — have different uses and produce different sequences from the same tree.

🗺️ Three Traversal Orders
  • In-order (Left, Root, Right): Visits BST nodes in sorted order
  • Pre-order (Root, Left, Right): Useful for copying/serializing a tree
  • Post-order (Left, Right, Root): Used for deletion, expression evaluation
# For BST with values inserted: 5, 3, 7, 1, 4 # 5 # / \ # 3 7 # / \ # 1 4 def inorder(node): # → 1, 3, 4, 5, 7 (sorted!) if node: inorder(node.left) print(node.val) inorder(node.right) def preorder(node): # → 5, 3, 1, 4, 7 if node: print(node.val) preorder(node.left) preorder(node.right)

BST with root 5, left child 3, right child 7. What does in-order traversal print first? Type the number.

Lesson 04 · CS Theory

Balanced Trees & Heaps

An unbalanced BST can degrade to O(n) — essentially a linked list. Balanced trees maintain height ≈ log n automatically. Heaps are another tree structure optimized for finding the minimum or maximum in O(1).

⚖️ Why Balance Matters

Insert 1, 2, 3, 4, 5 into a BST in order — you get a straight line to the right. Search is O(n). Insert them in random order and you get a balanced tree with O(log n) search. Self-balancing trees (AVL, Red-Black) fix this automatically.

🏔️ Min-Heap Property

In a min-heap, every parent is smaller than its children. The minimum is always at the root — O(1) access. Insert and delete are O(log n). Python's heapq module gives you a min-heap for free.

import heapq h = [] heapq.heappush(h, 5) heapq.heappush(h, 1) heapq.heappush(h, 3) print(h[0]) # 1 — minimum is always at index 0 heapq.heappop(h) # removes and returns 1 print(h[0]) # 3 — next minimum

File Systems
Your hard drive's folder structure is an N-ary tree. Every path from / to a file traces a root-to-leaf path.
🌐
HTML DOM
Every webpage is a tree — html → body → div → p. Browser rendering engines traverse this tree millions of times per second.
🗄️
Database B-Trees
Most database indexes use B-trees — balanced trees optimized for disk reads. Enables O(log n) queries on billions of rows.
🎯
Priority Queues
Heaps power priority queues used in Dijkstra's algorithm, OS scheduling, and A* pathfinding.
