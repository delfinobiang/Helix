Lesson 01

Variables & Expressions

In math, a variable is a letter that stands in for a number we don't know yet — like x or y. In programming, a variable is a named box that holds a value. They're the same idea, just written differently.

💡 Math vs Code — same concept, different syntax

Math writes y = 2x + 5.
Python writes y = 2 * x + 5.

Both say: "whatever x is, multiply it by 2 and add 5 to get y."

An expression is a combination of variables, numbers, and operations that produces a value. It's not a complete sentence — it doesn't say "equals something". It's just a value waiting to be computed.

3x² + 2x − 5

This expression has three terms: 3x², 2x, and −5. The numbers in front of the variables (3 and 2) are called coefficients. The −5 with no variable is a constant.

Lesson 02

Equations & Solving

An equation is different from an expression — it has an equals sign. It's a statement that two things are equal. When you solve an equation, you find the value of the unknown variable that makes the statement true.

⚖️ The Golden Rule of Equations

Whatever you do to one side of the equals sign, you must do to the other. Think of it as a balance scale — both sides must always stay balanced.

Let's solve 2x + 6 = 14 step by step:

STEP 1 Subtract 6 from both sides to isolate the term with x
2x + 6 − 6 = 14 − 6  →  2x = 8
STEP 2 Divide both sides by 2 to isolate x
2x ÷ 2 = 8 ÷ 2  →  x = 4
CHECK Plug x = 4 back in: 2(4) + 6 = 8 + 6 = 14 ✓

Equation Solver — ax + b = c

ax + b = c
# Solving ax + b = c in Python def solve_linear(a, b, c): # x = (c - b) / a if a == 0: return "No solution (a cannot be 0)" return (c - b) / a print(solve_linear(2, 6, 14)) # → 4.0 print(solve_linear(3, -9, 0)) # → 3.0 print(solve_linear(5, 0, 20)) # → 4.0

Functions & Graphs

A function is a machine: you put a number in, it does something to it, and spits a number out. Every input gives exactly one output. In math it's written f(x) — in Python it's literally a def.

Function Machine — f(x) = 2x + 1

x = 3
f(3) = 2×3 + 1 = 7
🐍 Functions in Python are identical in spirit

Math's f(x) = 2x + 1 translates directly to Python.
When you call f(3), Python computes 2*3 + 1 = 7 — exactly what the math says.

# f(x) = 2x + 1 in Python def f(x): return 2 * x + 1 print(f(3)) # → 7 print(f(0)) # → 1 print(f(-4)) # → -7 # You can also map it across a list! xs = [1, 2, 3, 4, 5] ys = [f(x) for x in xs] print(ys) # → [3, 5, 7, 9, 11]

A graph is a visual picture of a function. Every point on the line is an (x, y) pair where y = f(x). The slope tells you how steep the line is — for f(x) = 2x + 1 the slope is 2 (go right 1 unit → go up 2 units). The y-intercept is where the line crosses the y-axis (when x = 0): here it's 1.

Lesson 04

Algebra in Code — Real Examples

Algebra isn't just for math class. You use it every day as a programmer — calculating positions, prices, scores, conversions, and animations. Here are real patterns you'll write again and again.

🎮
Game Score
score = base + bonus * level
Classic linear expression — the score grows by a constant amount each level.
🌡️
Temperature Convert
C = (F − 32) × 5/9
Directly solving an algebraic equation: rearrange the formula to isolate C.
📏
Centering an Element
x = (parent_w − child_w) / 2
Every UI framework uses this to centre elements on screen.
💸
Discount Price
final = price × (1 − discount)
E-commerce: factored-out algebra to apply percentage discounts cleanly.
🎞️
Animation Lerp
pos = start + (end − start) × t
Linear interpolation — the algebraic backbone of smooth animations (t goes 0→1).
📈
Compound Interest
A = P × (1 + r)^n
Exponential growth formula — applies to finance, population, and viral spread.
# Real algebra in action: animation lerp def lerp(start, end, t): """Linear interpolation between start and end. t=0 returns start, t=1 returns end, t=0.5 returns midpoint.""" return start + (end - start) * t # Move a sprite from x=0 to x=400 over 10 frames for frame in range(11): t = frame / 10 x = lerp(0, 400, t) print(f"Frame {frame}: x = {x}") # Temperature converter def fahrenheit_to_celsius(F): return (F - 32) * 5 / 9 print(fahrenheit_to_celsius(212)) # → 100.0 (boiling) print(fahrenheit_to_celsius(32)) # → 0.0 (freezing)
