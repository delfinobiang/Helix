Sets & Set Operations

A set is an unordered collection of distinct objects. Sets are one of the most fundamental structures in mathematics — and they show up everywhere in programming: arrays, hash sets, databases, type systems.

📦 What is a Set?

A set contains unique elements with no duplicates and no defined order. We write sets with curly braces: A = {1, 2, 3}. The empty set is written ∅ or {}.

💡 Sets in Python

Python has a built-in set type with these exact operations: | for union, & for intersection, - for difference. a = {1,2,3}; b = {2,3,4}; a | b → {1, 2, 3, 4}

Relations & Functions

A relation describes a connection between elements of sets. A function is a special relation where every input has exactly one output — the mathematical foundation of every function you write in code.

📌 Functions as Mappings

A function f: A → B maps each element of set A (the domain) to exactly one element of set B (the codomain). The set of actual outputs is called the range (or image).

PropertyMeaningExample
DomainSet of all valid inputsf(x) = √x → domain: x ≥ 0
CodomainSet the function maps intof: ℝ → ℝ
Range (Image)Set of actual outputsf(x) = x² → range: y ≥ 0
Injective (one-to-one)Different inputs → different outputsf(x) = 2x
Surjective (onto)Every output has at least one inputf(x) = x³
BijectiveBoth injective and surjectivef(x) = x + 1 over ℤ
🔑 Why Bijections Matter in CS

Hash functions aim to be bijective (no collisions). Encryption needs bijections — each plaintext maps to exactly one ciphertext and vice versa. Database primary keys are injective: every row has a unique ID.

# Python: functions are mathematical mappings f = {1: 'a', 2: 'b', 3: 'c'} # dict = mathematical function # domain: keys, range: values domain = set(f.keys()) # {1, 2, 3} rng = set(f.values()) # {'a', 'b', 'c'} # injective test: no two keys map to same value injective = len(rng) == len(domain) print(injective) # True — it's injective

🌳 Trees are Special Graphs

A tree is a connected graph with no cycles. It has exactly n−1 edges for n vertices. Binary search trees, file systems, DOM — all trees. Every tree is a graph, but not every graph is a tree.

# Adjacency list — most common graph representation graph = { 'A': ['B', 'C'], 'B': ['A', 'D'], 'C': ['A', 'D'], 'D': ['B', 'C'], } # BFS — find shortest path from collections import deque def bfs(graph, start, target): queue, visited = deque([[start]]), {start} while queue: path = queue.popleft() node = path[-1] if node == target: return path for n in graph[node]: if n not in visited: visited.add(n) queue.append(path + [n]) print(bfs(graph, 'A', 'D')) # ['A', 'B', 'D']

Combinatorics

Combinatorics is the mathematics of counting. How many passwords are possible? How many ways can you seat 5 people? How many test cases cover all combinations? These questions all have precise answers.

📐 The Fundamental Counting Principle

If event A can happen in m ways and event B can happen in n ways, then both can happen in m × n ways. A 3-digit PIN with digits 0–9 has 10 × 10 × 10 = 1,000 possibilities.

OperationFormulaMeaningOrder matters?
Factorial n!n × (n−1) × … × 1Ways to arrange n itemsYes
Permutation P(n,r)n! / (n−r)!r items from n, orderedYes
Combination C(n,r)n! / (r! × (n−r)!)r items from n, unorderedNo
P(n,r) = n! / (n−r)!     C(n,r) = n! / (r! · (n−r)!)

Mathematical Induction & Logic

Mathematical induction is how we prove statements hold for all natural numbers. It's the mathematical equivalent of a recursive algorithm — prove the base case, then prove the inductive step.

🔁 The Two Steps of Induction
  • Base case: Prove the statement is true for n = 1 (or n = 0)
  • Inductive step: Assume it's true for n = k (inductive hypothesis), then prove it's true for n = k + 1
📋 Classic Example: Sum Formula

Prove: 1 + 2 + 3 + … + n = n(n+1)/2

Base (n=1): 1 = 1(2)/2 = 1 ✓

Inductive step: Assume 1+…+k = k(k+1)/2. Then 1+…+k+(k+1) = k(k+1)/2 + (k+1) = (k+1)(k+2)/2 ✓

Propositional Logic — Quick Review

ConnectiveSymbolTrue when…
AND (conjunction)P ∧ QBoth P and Q are true
OR (disjunction)P ∨ QAt least one is true
NOT (negation)¬PP is false
ImplicationP → QIf P then Q (false only when P=T, Q=F)
BiconditionalP ↔ QP and Q have the same value
🔁 Induction = Recursion

Induction and recursion are the same idea. In induction: base case + inductive step. In recursion: base case + recursive call. Every recursive proof has a matching inductive proof.
