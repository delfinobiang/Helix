Lesson 01 · CS Theory

Why Sorting Matters

Sorting is one of the most studied problems in computer science. A sorted array unlocks binary search — turning O(n) lookups into O(log n). Nearly every database query, search engine ranking, and recommendation system relies on sorted data.

⚡ The Power of Sorted Data

Finding an element in an unsorted list of 1 billion items takes up to 1 billion steps. In a sorted list, binary search finds it in just 30 steps. Sorting once pays off forever for every search that follows.

Lesson 02 · CS Theory

Bubble & Selection Sort

The simplest sorting algorithms. Both run in O(n²) — fine for small arrays, disastrous at scale. Understanding them builds intuition for why faster algorithms exist.

🫧 Bubble Sort

Repeatedly compare adjacent elements and swap them if out of order. Each pass "bubbles" the largest unsorted element to its correct position. After n passes, the array is sorted.

def bubble_sort(arr): n = len(arr) for i in range(n): swapped = False for j in range(n-i-1): if arr[j] > arr[j+1]: arr[j], arr[j+1] = arr[j+1], arr[j] swapped = True if not swapped: break # already sorted! return arr

Merge Sort

Merge sort uses divide and conquer: recursively split the array in half, sort each half, then merge the sorted halves. Guaranteed O(n log n) — always, no bad cases.

✂️ Divide and Conquer
  • Divide: Split array into two halves
  • Conquer: Recursively sort each half
  • Combine: Merge two sorted halves into one sorted array
def merge_sort(arr): if len(arr) <= 1: return arr mid = len(arr) // 2 left = merge_sort(arr[:mid]) right = merge_sort(arr[mid:]) return merge(left, right) def merge(left, right): result = [] i = j = 0 while i < len(left) and j < len(right): if left[i] <= right[j]: result.append(left[i]); i += 1 else: result.append(right[j]); j += 1 return result + left[i:] + right[j:]

Lesson 04 · CS Theory

Quicksort

Quicksort picks a pivot, partitions elements around it (smaller left, larger right), and recursively sorts each partition. Average O(n log n), uses O(log n) space — often faster in practice than merge sort.

🎯 The Pivot Strategy

The choice of pivot affects performance. Random pivot avoids worst-case O(n²). "Median of three" (first, middle, last elements) is a common optimization. Python's Timsort uses insertion sort for small arrays and merge sort for large ones.

def quicksort(arr): if len(arr) <= 1: return arr pivot = arr[len(arr) // 2] left = [x for x in arr if x < pivot] middle = [x for x in arr if x == pivot] right = [x for x in arr if x > pivot] return quicksort(left) + middle + quicksort(right) # Python's built-in sort is even better: arr.sort() # in-place Timsort O(n log n) sorted(arr) # returns new sorted list
